# 关注榜硬筛策略优化方案 2026-05-31

## 结论

当前策略的交易思想是成立的：策略1抓“趋势中继回踩后再启动”，策略2抓“突破共振后的强确认”，策略3作为“多因子主升预备池”补充样本。但当前最大问题不是规则不清晰，而是“硬筛条件太多、阈值太固定、实时路径和回测路径不完全一致”，导致好票容易被漏掉，模型也容易在少样本上产生概率两极分化。

更专业的方向不是简单放松所有条件，而是把策略系统拆成三层：

1. 硬约束层：只保留不可妥协的交易安全条件，例如主板非ST、流动性下限、非末端高拥挤、非破位。
2. 候选评分层：把涨幅、均线、回踩、板块、资金、分时、消息、启动质量改成连续分数，而不是全部用一刀切阈值。
3. 交易决策层：按策略类型分别预测和校准，策略1偏3到5日，策略2偏1到3日，策略3偏发现早期主升预备。

这套改法的目标是：不牺牲风控下限，同时扩大可学习样本，让模型从“少量硬筛票里排序”升级为“全市场宽候选池里识别真正有收益/风险比的票”。

## 当前实现的主要证据

当前生产候选池主要在 `src/a_share_predictor/store.py` 中实现：

- 策略1硬筛要求 `15日涨幅 10%~30%`、`价格 > MA5 > MA10 > MA20`、`回调3~6日`、`量缩`、`守MA10`、`当日涨幅 > 2%`、`成交额 > 2亿`、`换手 > 3%`、`20日涨幅 < 35%`。
- 策略2硬筛要求 `当日涨幅 > 5%`、`3日涨幅 > 10%`、`5日涨幅 > 15%`、`成交额 > 3亿`、`换手 > 5%`、`10日新高/接近新高`、`10日最大涨幅 < 40%`、`板块强势且上涨家数 >= 3`。
- 策略3已经引入 `long_setup_quality`、`resonance_quality`、`launch_readiness`、`breakout_quality`、`crowding_risk`、`risk_of_late_entry` 等更接近连续评分的变量。

当前全市场回测候选池主要在 `src/a_share_predictor/market_backtest_runner.py` 中实现，但它的策略3是简化版，只用 `launch_score / resonance_score / quality_score / late_risk` 等临时计算，和实时路径中的完整特征并不完全一致。这会造成一个重要问题：页面候选池和全市场回测候选池不是同一个“策略生物”，模型迭代会被噪声干扰。

已有本地证据也支持“不能盲目扩大榜单”：

- `docs/candidate_breadth_2026-05-28.md` 显示当前可回放切片中，top3 的年化和回撤效率优于更宽篮子；top10 虽然容量增加，但回撤明显放大。
- `docs/ten_year_strategy_optimization_report_2026-05-28.md` 显示，真正有效的不是策略单独硬筛，而是“策略候选池 + 模型分数阈值 + 市场状态过滤 + 空仓机制”的组合。
- `docs/default_path_evidence_gap_2026-05-28.md` 显示，当前默认 h3/10% 路径有可观测性和模型源不一致问题，不能继续把它当成完整健康基线来过度优化。

## 为什么硬筛会漏掉好票

### 1. 多个“必要条件”实际是相关条件

策略1里 `15日涨幅`、`均线多头`、`当日涨幅`、`回调未破MA10` 都在描述同一个主题：趋势中继。它们同时硬筛，会把同一类信号重复卡门槛。

典型漏票：

- 慢牛刚启动：15日涨幅只有 6% 到 9%，但量价结构很好，被 `10%` 下限漏掉。
- 强势浅调：只调整 1 到 2 天就重新上攻，被 `3~6日调整` 漏掉。
- 机构趋势票：成交额足够但换手率不高，被 `换手 > 3%` 漏掉。
- 高价低换手核心票：趋势质量高，但换手率天然偏低，被小盘股口径误伤。

### 2. 固定阈值没有适配市场状态

同样的 `今日涨幅 > 2%` 在牛市、震荡市和弱市意义不同：

- 强趋势市场：2% 可能只是普通确认，阈值可以提高并追求更强板块共振。
- 震荡市场：2% 可能已经是较强信号，但需要更强分时承接确认。
- 缩量弱市：2% 可能是诱多，必须提高收盘位置和资金净流入要求。

当前策略缺少“市场状态 -> 阈值映射”，所以会出现某些日子策略1/2完全选不出股票，或者某些强行情中错把末端加速当启动。

### 3. 策略1和策略2混用同一预测目标

策略1是“回踩后的中继”，更适合 3日/5日收益风险比；策略2是“强势突破确认”，更适合 1日/3日延续概率。两者如果使用同一套预测窗口、同一概率口径，会出现：

- 策略1的低波动优势被低估。
- 策略2的隔日冲高回落风险被低估。
- 模型把“稳健中继”和“短线强攻”混成一种标签，解释力下降。

### 4. 过多硬筛会让模型样本变少

模型最怕的不是候选池宽，而是训练样本“窄且偏”。如果只让模型看到极少数硬筛后的股票，模型学到的不是全市场规律，而是筛选器的偏见。

表现会是：

- 概率两极分化。
- 样本外命中率不稳定。
- 某些交易日没有策略1/2样本，只能靠策略3补位。
- 回测结果对少数年份、少数板块过度敏感。

### 5. 页面路径和回测路径不一致

实时路径的策略3已经有更完整的多因子质量评分，而全市场回测路径仍是较简化的 fast candidate builder。这个差异会导致：

- 回测不能完整评估页面真实选股逻辑。
- 参数优化可能优化到“回测简化策略”，而不是“生产真实策略”。
- 归因时无法判断收益来自结构、资金、分时、板块还是模型排序。

## 应保留为硬筛的条件

硬筛只保留“错了会产生不可接受风险”的条件。

### 全策略通用硬筛

- 主板非ST，排除退市风险和明显异常状态。
- 当日可交易，非停牌，非一字涨停后不可买，非严重流动性不足。
- 成交额下限：默认不低于 1亿，弱市可提高到 1.5亿，策略2可提高到 2亿。
- 价格未明显破位：收盘价不低于 MA20 的 97% 或不低于关键平台下沿。
- 末端风险过滤：20日涨幅、10日最大涨幅、连续放量长上影、极端换手同时出现时否决。
- 数据质量过滤：最新交易日、复权日线、成交额、换手率、行业字段必须可用。

### 策略1硬筛

策略1只保留这些硬约束：

- 价格在 MA10 上方或最多轻微跌破后快速收回。
- MA5、MA10、MA20 大体多头，允许小幅粘合，不要求严格 `MA5 > MA10 > MA20`。
- 最近20日涨幅不处于明显末端，例如不超过 40% 到 45%。
- 回调期不能放量破位，不能出现连续大阴线。

原来的 `15日涨幅 10%~30%`、`回调3~6日`、`今日涨幅 > 2%`、`换手 > 3%` 应改为评分项。

### 策略2硬筛

策略2只保留这些硬约束：

- 当日强度必须明显高于市场，例如个股涨幅超过全市场中位数/行业中位数至少 2 到 3 个百分点。
- 当日收盘位置不能太差，例如收在日内区间 55% 以上。
- 接近10日新高或突破平台。
- 板块不是孤立上涨，至少有板块广度或资金共振之一。
- 排除末端加速：高换手、长上影、连续大涨、远离MA5过多同时出现时否决。

原来的 `今日涨幅 > 5%`、`3日 > 10%`、`5日 > 15%`、`换手 > 5%` 应改为强度评分，而不是绝对硬门槛。

## 应改成软评分的条件

### 趋势结构分

建议分数：

```text
 MA5/MA10/MA20 多头或粘合上拐
 收盘价站上 MA10 / MA20
 15日涨幅处于 6%~28% 的甜蜜区
 20日涨幅未明显过热
 平台突破或回踩平台不破
 波动收敛后放量启动
 远离均线过大扣分
 长上影和高位放量滞涨扣分
```

### 回踩承接分

建议不要再用“必须调整3~6日”，而是评分：

```text
 调整1~2日：强势浅调，加分但要求分时承接强
 调整3~6日：标准中继，加分最高
 调整7~10日：需要平台不破和缩量，否则降分
 调整期成交量低于启动日均量 40%~75%：加分
 调整期跌破MA10但快速收回：小幅扣分，不直接淘汰
 调整期跌破MA20或平台下沿：否决
```

### 板块共振分

目前“板块内至少3只上涨”太弱，应升级为：

```text
 板块近2日涨幅排名分位
 板块上涨家数占比
 板块成交额相对5日/20日放大
 板块内涨幅前排数量
 个股在板块内强度排名
 板块是否连续2日强于全市场
 个股是否跟随板块还是明显落后板块
```

硬要求只保留“不能完全孤立”，其余交给评分。

### 资金分

资金不建议简单硬卡主力净流入，因为 A 股资金口径噪声很大。更合理的是：

```text
 今日成交额分位
 成交额相对20日均值倍数
 量价同向程度
 主力资金近1/3/5日净流入斜率
 开盘30分钟资金承接
 尾盘30分钟资金回流
 高换手但收盘弱：扣分
 放量突破且收盘强：加分
```

### 分时分

分时应作为“买点确认”和“假突破过滤”，而不是首轮硬筛唯一依据：

```text
 开盘冲高后回落幅度
 日内低点出现时间
 收盘价相对日内高低区间位置
 尾盘30分钟收益
 VWAP上方运行时间占比
 高点是否出现在尾盘
 放量时价格是否同步上行
```

### 消息面分

消息面应该主要做事件修正，而不是强制选股：

```text
 产品/技术/订单类利好：短期加分
 股东减持、监管风险、事故风险：扣分或否决
 业绩类消息：按方向和预期差评分，不简单看“利好/利空”
 热点舆情：只在板块资金同步时加分，孤立新闻不加大权重
```

已有新闻研究显示，不同事件类别的1/3/5日表现差异明显，消息面应以事件类别和方向做分策略校准，而不是只作为文本展示。

## 新版策略结构

### 策略1：趋势中继回踩启动

定位：抓主升刚开始或第一段主升后的健康回踩。

硬筛：

```text
 主板非ST
 成交额 >= 自适应流动性下限
 close >= MA20 * 0.97
 20日涨幅 < 市场状态自适应上限
 未出现放量破位
```

软评分：

```text
 trend_score = 均线多头/粘合上拐 + close_vs_ma10 + close_vs_ma20
 pullback_score = 调整天数甜蜜区 + 缩量质量 + 平台不破
 launch_score = 今日涨幅 + 收盘位置 + 尾盘强度 + 放量合理性
 sector_score = 板块强度 + 板块广度 + 个股板块内排名
 risk_penalty = 高位拥挤 + 长上影 + 放量滞涨 + 跌破关键位
```

候选条件：

```text
 strategy1_score >= 62
 risk_penalty < 35
```

预测口径：

- 主预测：3日和5日平均持有收益。
- 辅助预测：1日低开/冲高回落风险。
- 页面解释：偏“中继低吸/确认买点”，不要和策略2混用同一进攻标签。

### 策略2：突破共振强确认

定位：抓板块主线里刚被市场确认的强票。

硬筛：

```text
 主板非ST
 成交额 >= 自适应高流动性下限
 收盘位置 >= 55%
 距10日高点 <= 3% 或突破平台
 末端拥挤风险未触发
```

软评分：

```text
 breakout_score = 今日相对强度 + 3/5日相对强度 + 新高质量
 sector_resonance = 板块2日排名 + 上涨家数占比 + 板块成交额扩张
 money_score = 成交额放大 + 主力净流入 + 尾盘回流
 intraday_quality = 收盘位置 + VWAP强度 + 尾盘强弱
 late_risk_penalty = 连续加速 + 高换手长上影 + 远离MA5/MA10
```

候选条件：

```text
 strategy2_score >= 66
 sector_resonance >= 55
 late_risk_penalty < 40
```

预测口径：

- 主预测：1日和3日收益。
- 重点风险：次日高开低走、冲高回落、板块退潮。
- 页面解释：偏“突破确认/短线强攻”，持有纪律要比策略1更快。

### 策略3：多因子主升预备池

定位：不让策略1/2过硬导致完全无样本，同时提前发现“还没完全满足策略1/2、但主升概率正在抬升”的股票。

硬筛：

```text
 主板非ST
 close > MA10 或 close > MA20 且 MA5上拐
 5日涨幅在 0%~18%
 20日涨幅在 -8%~35%
 成交额 >= 8000万到1.2亿，按市场状态自适应
 风险末端未触发
```

软评分沿用并强化当前实时路径中的：

```text
 long_setup_quality
 resonance_quality
 launch_readiness
 breakout_quality
 board_resonance_strength
 quant_score
 crowding_risk
 risk_of_late_entry
```

候选条件：

```text
 strategy3_score >= 56 到 60，自适应调整
```

预测口径：

- 主预测：3日/5日潜在收益。
- 页面解释：偏“主升预备/观察确认”，不能和策略2一样直接强攻。

## 自适应阈值方案

不要用固定阈值一套打全市场。建议按市场状态动态调整：

| 市场状态 | 候选池目标 | 策略1阈值 | 策略2阈值 | 策略3阈值 | 仓位建议 |
|---|---|---:|---:|---:|---|
 强趋势 | 提高进攻质量 | 64 | 66 | 58 | 可开仓 |
 震荡偏强 | 平衡机会和风险 | 61 | 68 | 58 | 小仓/精选 |
 震荡偏弱 | 降低追高，保留观察 | 63 | 72 | 61 | 轻仓/观察 |
 弱势 | 不强行选股 | 68 | 75 | 65 | 默认空仓 |

同时设置每日候选数量目标：

```text
 hard_pool_min = 30
 hard_pool_target = 80~150
 display_top = 50
 trade_top = 3~5
```

如果策略1/2不足，不应该降低风控硬约束，而应该由策略3补足“观察池”；交易层仍然只买模型和市场状态都通过的前排。

## 模型优化方案

### 1. 分策略建模或分策略校准

最低成本做法：

```text
 统一模型预测全市场
 + strategy_id 作为特征
 + strategy-specific calibration
 + strategy-specific horizon
```

更理想做法：

```text
 策略1：3/5日收益回归 + 达标概率
 策略2：1/3日收益回归 + 回撤风险
 策略3：3/5日收益回归 + 主升转化概率
```

不要只预测“是否达到10%”。短周期10%标签太稀疏，会导致概率不稳定。建议同时预测：

```text
 forward_return_1d
 forward_return_3d
 forward_return_5d
 max_high_return_3d
 max_drawdown_3d
 hit_3pct_3d
 hit_5pct_5d
```

页面主显示可以继续是“达标概率”，但模型训练应该同时看收益、回撤、冲高、胜率。

### 2. 标签从固定收益改为收益风险比

对交易系统更有用的目标不是单纯上涨概率，而是：

```text
 expected_edge = E[forward_return] - cost - risk_penalty
 rr_score = expected_upside / expected_drawdown
 trade_label = forward_return_3d > max(市场基准, 成本 + 风险补偿)
```

这样可以避免模型偏好“涨幅大但回撤也大”的票。

### 3. 引入负样本和错过样本

为了减少漏票，训练集不能只包含最终关注榜，还要包含：

```text
 策略1/2/3命中的股票
 差一点命中策略1/2的股票
 同行业强势但未入选股票
 全市场随机负样本
 当日排名50~300的边缘样本
```

重点学习“为什么某些未入选股票后来涨了”，这比只学习已入选股票更能减少漏票。

### 4. 避免过拟合

必须执行这些约束：

- 使用按时间切分的 walk-forward，而不是随机切分。
- 对未来收益标签有重叠的样本，使用 purged/embargo 思路，避免标签泄漏。
- 每次参数搜索记录 trial count，禁止无限调参后只汇报最优结果。
- 使用统一 portfolio NAV 回测，不再只看平均收益。
- 同时看成本压力，至少 0bp/10bp/20bp/30bp。
- 对每个策略单独输出样本量、胜率、平均收益、最大回撤、分年表现。
- 引入 PBO/DSR 思路：参数搜索越多，收益门槛越高。

外部量化研究也支持这一点：Bailey、Borwein、López de Prado 和 Zhu 的 PBO/CSCV 框架强调，策略选择中的多次回测会系统性放大样本内表现；Bailey 和 López de Prado 的 Deflated Sharpe Ratio 也是为修正多重测试和非正态收益造成的表现膨胀。

## 回测优化方案

### 1. 统一生产候选池和回测候选池

第一优先级是把 `store.py` 中的完整候选构建能力抽到共享模块，让页面和回测都调用同一套策略特征。

目标：

```text
 production_candidate_builder == backtest_candidate_builder
```

不能让页面用完整策略3，回测用简化策略3。

### 2. 回测必须按点时间数据

每个历史交易日只能使用当日收盘前后真实可得的数据：

- 日线结构：使用复权历史数据判结构。
- Tushare：只做非复权截面补充，不用于复权结构判断。
- 板块和资金：用当日可得快照或当日之后不可见字段禁用。
- 新闻：按发布时间切分，不能用未来公告。
- 股票池：使用历史当日可交易股票，避免幸存者偏差。

### 3. 输出三套榜单指标

每天回测输出：

```text
 strategy1_count
 strategy2_count
 strategy3_count
 selected_count
 top3_return_1d / 3d / 5d
 top5_return_1d / 3d / 5d
 avg_drawdown_3d
 hit_rate_3pct
 turnover
 cost_adjusted_return
 market_state
```

这样才能知道是策略没选到，模型排序差，还是市场状态不适合交易。

### 4. 回测分层报告

必须按以下维度分层：

- 策略来源：策略1、策略2、策略3。
- 市场状态：强趋势、震荡、弱势。
- 板块强度：top行业、中位行业、弱行业。
- 流动性：成交额分位。
- 位置：低位启动、中位突破、高位加速。
- 消息：利好、利空、中性、无消息。

没有分层，平均收益会掩盖真正的问题。

## 页面与产品层优化

页面不应只显示“策略来源”，还应显示：

```text
 策略来源：策略1/策略2/策略3
 策略评分：strategy_score
 核心加分：趋势/回踩/板块/资金/分时/消息
 核心扣分：拥挤/末端/上影/放量滞涨/弱板块
 预测窗口：1日/3日/5日
 适配交易方式：低吸确认/突破确认/观察池
 是否可交易：由市场状态 + 模型分数 + 执行分共同决定
```

如果策略1/2为空，需要提示用户，但提示不应等同于“策略失效”。更准确的解释是：

```text
 今日严格策略1/2未出现足够高质量交易型样本，关注榜由策略3主升预备池补位；策略3用于观察和模型排序，不代表策略1/2买点已经出现。
```

## 落地顺序

### P0：先统一候选池

1. 抽象 `candidate_rules.py`，把策略1/2/3的特征、硬筛、软评分、否决原因统一放进去。
2. `store.py` 和 `market_backtest_runner.py` 都调用同一套候选构建函数。
3. 每个候选保留 `hard_gate_pass`、`soft_score`、`veto_reason`、`score_breakdown`。

验收：

- 同一交易日、同一数据输入，页面候选池和回测候选池策略标签一致。
- 策略1/2/3数量一致。
- 测试覆盖“策略1/2为空时策略3补位但不冒充硬买点”。

### P1：把硬筛改成硬约束 + 软评分

1. 策略1放宽 `15日涨幅`、`调整天数`、`换手率`。
2. 策略2放宽 `3日/5日涨幅` 的硬条件，改为相对强度分。
3. 板块上涨家数从绝对数量改为“上涨占比 + 板块排名 + 板块成交额扩张”。
4. 所有阈值按市场状态自适应。

验收：

- 每日宽候选池稳定在 80 到 150 只附近。
- 策略1/2不再频繁为0。
- display top50 仍由模型和风险过滤筛出，不直接扩大交易票数量。

### P2：分策略预测和校准

1. 给模型加入 `candidate_strategy`、`strategy_score`、`score_breakdown`。
2. 策略1主看3/5日，策略2主看1/3日，策略3主看3/5日转化。
3. 页面同时展示收益预测、回撤风险和达标概率。

验收：

- 每个策略单独输出 Brier、AUC、top bucket return、平均收益。
- 概率分布不再大幅两极化。
- 策略2的次日冲高回落风险能被显式扣分。

### P3：全市场滚动回测与防过拟合

1. 半年、三年、十年三套回测同时保留。
2. 半年用于近期适配，三年用于市场周期，十年用于鲁棒性。
3. 参数搜索使用固定 search budget，并记录所有失败参数。
4. 使用统一 portfolio NAV 和成本压力。

验收：

- 新策略必须在 10bp/20bp/30bp 成本下分别输出表现。
- 至少按年份和市场状态证明不是单一年份贡献。
- 没有足够样本时不允许提高参数复杂度。

## 推荐的最终系统形态

```text
 全市场历史可交易股票
 -> 主板非ST/数据质量/流动性/末端风险硬约束
 -> 策略1/2/3宽候选识别
 -> 趋势/结构/板块/资金/分时/消息连续评分
 -> 分策略模型预测 1/3/5日收益、回撤和达标概率
 -> 市场状态过滤和仓位建议
 -> top50关注榜展示
 -> top3/top5交易级候选
 -> 每日轻量回测 + 定期全市场滚动回测
 -> 根据真实回测更新校准层，而不是随意调硬阈值
```

## 最重要的执行纪律

1. 不要因为策略1/2某天为空就强行降低所有硬阈值。
2. 不要因为策略3能补足数量，就把策略3当成策略1/2的买点替代。
3. 不要用关注榜样本单独训练模型，必须保留全市场负样本和边缘样本。
4. 不要只看平均收益，要看真实组合净值、最大回撤、成本压力和分年稳定性。
5. 不要继续让页面候选池和回测候选池使用不同规则。

一句话：你的策略要从“硬筛出我认为像主升的票”，升级为“先宽召回所有可能主升的票，再用模型和风险收益比筛掉伪机会”。这才是既不漏好票，又不被噪声骗进去的专业化路线。
